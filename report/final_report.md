# When Machine Learning Fails: Diagnosing and Repairing Shortcut Learning in Online Shoppers

## Abstract

This mini-project studies a deliberately induced failure mode in a non-linear classifier trained on the UCI Online Shoppers Purchasing Intention dataset. The model is intentionally forced to fail under a deployment-like temporal shift: it is first allowed to learn shortcut correlations from calendar and acquisition proxies, then evaluated on late-year sessions where those correlations are less reliable. The analysis follows a research structure: symptom characterization, falsifiable causal hypothesis, controlled intervention, targeted correction, validation with uncertainty, and threats to validity.

The central claim is deliberately modest: the experiments do not prove the real-world causes of online purchases. They show that the prediction function can become brittle when it relies on non-invariant shortcut variables, and that blocking this failure path improves robustness under the project protocol.

## 1. Research Question and Chosen Dataset

Dataset: UCI Online Shoppers Purchasing Intention, a binary classification task where the target is whether a web session ends in purchase.

Research question: does a non-linear classifier trained on Online Shoppers sessions learn a spurious shortcut from calendar and acquisition proxies (`Month`, `TrafficType`, `VisitorType`, device/browser fields), such that it appears acceptable under an IID random split but loses robustness on late-year deployment months? If yes, can an invariant behavioral representation plus temporally adjacent validation improve out-of-period robustness without relying on that shortcut?

This question is falsifiable. The shortcut hypothesis would be weakened if:

- the shortcut-only model generalized as well as the behavioral model on Oct-Dec sessions;
- suspected shortcut variables showed no meaningful train-deployment drift;
- prediction-function interventions on shortcut variables did not change model behavior;
- permutation importance did not implicate shortcut variables.

## 2. Reference Model and Observed Symptom

The deliberately broken model is a Random Forest, satisfying the non-linear model-family constraint. It is restricted to shortcut-like variables: month, special-day proximity, operating system, browser, region, traffic type, visitor type, and weekend.

Two evaluation regimes are compared:

- random split, where seasonal regimes are mixed across train and test;
- temporal deployment split, where the model is trained on sessions up to September and evaluated on October-December.

The model was intentionally forced to fail under deployment-like temporal shift. Random validation leaks seasonal regimes across train and test, so it can overstate robustness. The symptom is characterized using PR-AUC, purchase recall, balanced accuracy, expected calibration error, Brier score, confusion matrices, and subgroup metrics by month.

## 3. Causal Hypothesis and Controlled Experiment

Causal hypothesis: the model relies on non-causal calendar/acquisition shortcuts. These variables are correlated with purchase in the observed training distribution but are not invariant causes of purchase behavior. When the temporal regime changes, the shortcut breaks.

The controlled experiment manipulates the representation while holding the model family, target, split logic, and metrics fixed:

- treatment expected to fail: shortcut-only representation;
- positive control: behavioral-intent representation;
- leakage sensitivity control: behavioral representation without `PageValues`;
- negative control: corrected representation excludes `Month`, so the shortcut path is blocked by design.

The intervention is causal with respect to the prediction function: the experiment changes what information the fitted model can use, not the real-world shopping process.

The causal evidence is triangulated:

- drift analysis verifies that suspected shortcut variables change between train and deployment regimes;
- controlled representation tests compare shortcut-only and behavioral models under the same temporal split;
- prediction-function interventions permute suspected variables while holding the trained model fixed;
- permutation importance checks whether the fitted model depends on shortcut variables;
- SHAP is included as an interpretability supplement, not as the main causal proof.

## 4. Proposed Correction and Evaluation

The correction targets the cause rather than merely masking the symptom:

- remove unstable calendar/acquisition proxy variables;
- train on behavioral intent features;
- select the threshold on September, a temporally adjacent validation month;
- use cost-aware class weighting and a recall-at-minimum-precision objective;
- report bootstrap uncertainty and seed variance.

This is not a generic SMOTE-style correction. The repair is tied to the failure mechanism: if shortcut reliance is the cause, blocking those variables and selecting decisions under a closer temporal regime should improve deployment robustness.

## 5. Bonus Failure Mode: Class Imbalance Hiding Purchase Failure

Symptom: purchases are the minority class, so a default threshold can produce superficially acceptable accuracy while failing to detect purchase sessions.

Hypothesis: the default decision threshold privileges majority-class behavior more than purchase detection.

Controlled test: keep the corrected model and predicted probabilities fixed; manipulate only the threshold rule.

Correction: choose a threshold on validation data with an explicit recall objective subject to a minimum precision constraint.

This failure mode is complementary to the main one. Representation repair addresses shortcut learning; threshold repair addresses asymmetric decision costs. The results are written to:

- `outputs/metrics/table12_bonus_threshold_failure.csv`;
- `outputs/metrics/table13_bonus_threshold_bootstrap_deltas.csv`.

## 6. Threats to Validity

Observational-data limitation: the experiments manipulate model inputs and validation protocols, not real user behavior. The causal claim is about the prediction function, not about the true causal drivers of purchase.

Coarse temporal axis: `Month` is not a true timestamp. The dataset is not a multi-year production log, so Oct-Dec deployment is a plausible stress test rather than a complete production replay.

Confounding between seasonality and campaigns: `Month`, `TrafficType`, and `VisitorType` may reflect promotions, traffic acquisition strategies, or external demand shocks. The project treats them as unstable proxies, not as intrinsically invalid variables.

Leakage risk: `PageValues` may encode in-session information close to the purchase event. The no-PageValues control checks whether the correction depends entirely on this potentially leaky signal.

Threshold dependence: the cost-aware threshold is linked to an explicit preference for purchase recall under a minimum precision constraint. Different business costs would justify a different threshold.

Statistical uncertainty: bootstrap intervals and seed variance reduce the risk of overinterpreting a single run, but they do not eliminate uncertainty. Some month-level subgroups remain small.

External validity: the conclusion should be retested on newer traffic logs before deployment. The current project demonstrates a failure mechanism, not a universal guarantee.

Model-family dependence: the analysis uses a tree ensemble. A neural network or gradient boosting model might learn a related but not identical shortcut profile.

## 7. Conclusion and What We Learned

The project demonstrates the intended scientific workflow:

- a model was intentionally forced to fail;
- the failure was made visible with metrics, confidence intervals, confusion matrices, subgroup analysis, calibration, and drift diagnostics;
- the proposed cause was tested through controlled representation changes and prediction-function interventions;
- the correction targeted the identified mechanism rather than applying a generic metric hack;
- uncertainty, leakage risk, and external validity limits were discussed explicitly.

The corrected model is not presented as perfect. The stronger and more defensible conclusion is that the original model failed for an experimentally diagnosable reason, and that a targeted intervention reduced that failure under a deployment-like temporal protocol.
